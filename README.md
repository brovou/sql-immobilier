## Français

## Contexte
Une agence souhaite structurer ses données (biens, ventes, caractéristiques) pour produire des indicateurs fiables et faciliter l’analyse.

## Objectif
Concevoir une base relationnelle et produire des requêtes SQL permettant de calculer des KPI (prix, volumes, répartition, performance), avec des contrôles de cohérence.

## Données
Données immobilières structurées (biens, ventes, localisation, caractéristiques).  
*(Aucune donnée sensible.)*

## Méthode
- Modélisation relationnelle : tables, clés, relations, contraintes
- Requêtes SQL : JOIN, agrégations, filtres, sous-requêtes
- Qualité : cohérence, doublons, intégrité référentielle

## Résultats (valeur)
- Base exploitable pour analyses récurrentes
- KPI reproductibles (prix moyen, volumes, distribution)
- Fiabilité améliorée (structure + contraintes)

## English

## Context
A real-estate organization needs structured data (properties, sales, attributes) to produce reliable indicators and support analysis.

## Objective
Design a relational database and write SQL queries to compute KPIs (prices, volumes, distribution, performance), including consistency checks.

## Data
Structured real-estate data (properties, sales, location, characteristics).  
*(No sensitive data.)*

## Method
- Relational modeling: tables, keys, relationships, constraints
- SQL queries: JOINs, aggregates, filters, subqueries
- Quality: consistency checks, duplicates, referential integrity

## Outcomes (value)
- Database ready for recurring analytics
- Reproducible KPIs (avg price, volumes, geographic distribution)
- Improved reliability (structure + constraints)
