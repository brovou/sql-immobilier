# Base de données immobilière (SQL)

## Contexte
Une agence immobilière souhaite structurer ses données (biens, ventes, caractéristiques) pour produire des indicateurs fiables et faciliter l’analyse.

## Objectif
Concevoir une base relationnelle et produire des requêtes SQL permettant de calculer des KPI (prix, volumes, répartition, performance), avec des contrôles de cohérence.

## Données
Données immobilières structurées (biens, ventes, localisation, caractéristiques).  
*(Aucune donnée sensible.)*

## Méthode
- Modélisation relationnelle : tables, clés, relations, contraintes
- Requêtes SQL : JOIN, agrégations, filtres, sous-requêtes
- Qualité : cohérence, doublons, intégrité référentielle

## Résultats (valeur)
- Base exploitable pour analyses récurrentes
- KPI reproductibles (prix moyen, volumes, distribution géographique, performance)
- Meilleure fiabilité des chiffres (structure + contraintes)

## Livrables
- Scripts SQL : `/src`
- Schéma / visuels : `/assets`
- Rapport (si applicable) : `/docs`

## Compétences mobilisées
SQL, modélisation, data quality, KPI, rigueur et reproductibilité.
